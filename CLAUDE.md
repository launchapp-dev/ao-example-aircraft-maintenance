# Aircraft Maintenance Pipeline — Agent Context

## What This Project Is

An automated maintenance management system for a small regional airline operating 6 DHC-8-400 (Q400) turboprop aircraft. The system handles the full maintenance lifecycle: daily flight-hour ingestion, inspection scheduling, work order generation, parts procurement, and post-maintenance airworthiness certification.

Aviation maintenance is safety-critical and governed by FAA 14 CFR Part 121 / EASA Part-M regulations. Every action must be traceable, every inspection interval respected, and every aircraft released to service by an authorized certifying engineer.

## Key Safety Rules (Agents Must Enforce)

1. **Inspection hard limits** — Aircraft must NOT fly past next-due inspection. Flag immediately when margin < 50 FH or < 7 days.
2. **MEL deferral limits** — Category B = 3 days, C = 10 days, D = 120 days. Never extend beyond category limit.
3. **Parts before scheduling** — Never create a work order unless parts are in stock or delivery confirmed before induction.
4. **CRS required** — No aircraft returns to service without a valid Certificate of Release to Service.
5. **Fleet availability** — Never schedule more than 2 aircraft for simultaneous C/D-check heavy maintenance.
6. **Snag triage** — Safety-critical snags (engine, flight controls, landing gear) ground the aircraft within 24 hours.
7. **Life limits** — A rotable at its TSO or total-time limit must be removed regardless of condition.

## Workflows

| Workflow | Trigger | Purpose |
|---|---|---|
| `daily-ops` | 06:00 daily | Ingest flight hours, triage snags, check inspection margins |
| `generate-work-order` | On-demand (after daily-ops flags schedule/urgent) | Create and validate maintenance work orders |
| `certify-release` | On-demand (after maintenance completion) | Issue airworthiness certificates, update records |
| `weekly-fleet-review` | 07:00 Monday | Fleet health report for management |

## Work Order Numbering

Format: `WO-YYYY-NNN` (e.g., `WO-2026-001`)
- Use current year and next sequential number
- Check existing files in `data/work-orders/` to determine next number

## CRS Numbering

Format: `CRS-YYYY-NNN` (e.g., `CRS-2026-001`)
- Use current year and next sequential number
- Check existing files in `data/airworthiness-certs/` to determine next number
